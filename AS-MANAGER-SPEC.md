## 1. 프로젝트 개요

- **프로젝트명**: AS Manager (콜라보 콤프레샤 AS 관리 시스템)
- **목적**: 기존 구글시트 3개(AS일지, 택배발송, 부속가격)를 하나의 웹사이트로 통합
- **사용자**: 2~3명이 하나의 사이트에서 각자 로그인하여 사용
- **특징**: AS 수리 하시는 분들이 많아서 폰트 크기를 크게 유지

---

## 2. 기술 스택

- **프론트엔드**: Next.js
- **데이터베이스**: Supabase (프로젝트 ID: dlzasdhhwgxshkyuzfyp, 서울 리전)
- **인증**: Supabase Auth (이메일/비밀번호)
- **파일 저장**: Supabase Storage (부품 이미지)
- **배포**: Vercel (GitHub 연동 자동 배포)
- **SMS**: httpSMS (안드로이드 폰 SMS 게이트웨이)
- **디자인**: 공구연구소 디자인 시스템 (Pretendard 폰트, #185FA5 프라이머리)

- **as_records** 테이블: 생성 완료, RLS 정책 설정됨

## 4. 페이지 구성 (5개 탭)

### 탭 1: AS 일지 (메인)

구글시트처럼 **한 행에서 입고→수리→입금→출고** 전체를 관리하는 통합 테이블.

#### 4.1.1 상단 KPI (5개)
- 전체 건수
- 접수/진단
- 수리중
- 완료
- 수리불가
- ※ 매출 관련 수치는 여기에 표시하지 않음 (설정→정산관리에서 확인)

#### 4.1.2 필터 (가로 한 줄 배치)
- 통합 검색창 (이름, 연락처, 모델, 증상 검색)
- 전체 구분 (AS 수리 / 제품 판매 / 부품 판매)
- 전체 상태 (접수 / 진단중 / 수리중 / 완료 / 수리X / 폐기)
- 전체 브랜드 (콜라보 / 마끼다 / 디월트 / 프레레)
- 월별 선택 (date month picker)

#### 4.1.3 테이블 컬럼 (컬러 그룹으로 구분)

**입고/고객/제품 (파란색 그룹)**:
| 컬럼 | 타입 | 비고 |
|------|------|------|
| 구분 | 드롭다운 | AS 수리 / 제품 판매 / 부품 판매 |
| 입고일 | 날짜 | 형식: 2026년 4월 4일 |
| 브랜드 | 드롭다운 | 콜라보 / 마끼다 / 디월트 / 프레레 / 기타 |
| 택배사 | 드롭다운 | 롯데택배/CJ대한통운/한진택배/경동택배/로젠택배/우체국/대신화물/방문/용차/퀵 |
| 운임 | 텍스트 | 예) 선불 4,000원, 착불 7,000원 |
| 계산서 | 드롭다운 | 없음(일반소매) / 계산서(거래처) / 월말 |
| 거래처/성함 | 텍스트 | 거래처명 + 고객명 (예: 신화종합 / 박병현) |
| 연락처 | 텍스트 | 010-0000-0000 |
| 모델명 | 드롭다운 | DC660/DC661/DC662/DC886/DC990X1/DC990K/DC990S/DC991/DC992/DC993/DC995/DC998/KOL-30AD/KOL-A20/DA25/DAC990X1/DW30AD/FVC-20C/기타 |
| 증상 | 텍스트 | 자유 입력 |
| 비고 | 텍스트 | 메모 |

**AS 처리 (초록색 그룹)**:
| 컬럼 | 타입 | 비고 |
|------|------|------|
| 처리결과 | 텍스트 | 예) 가스켓4SET, 메인보드 교체 |
| 처리자 | 드롭다운 | 풀네임 (설정에서 관리, 예: 최영호, 김영수) |
| AS상태 | 드롭다운 | 접수/진단중/부품대기/수리중/완료/수리X/폐기 |
| AS비용 | 숫자 | 콤마 구분자, 예) 75,000 |

**입금 (노란색 그룹)**:
| 컬럼 | 타입 | 비고 |
|------|------|------|
| 입금상태 | 드롭다운 | 완료/대기/명세서/무상/카드/방문결제 |
| 입금자 | 텍스트 | |

**출고 (보라색 그룹)**:
| 컬럼 | 타입 | 비고 |
|------|------|------|
| 출고일 | 날짜 | 형식: 2026년 4월 4일 |
| 택배사 | 드롭다운 | 입고 택배사 목록 + 매장 |
| 운송장번호 | 텍스트 | |
| 비고 | 텍스트 | 출고 메모 |
| 문자 | 아이콘 | 초록점(발송완료) / 회색점(미발송) |

#### 4.1.4 인라인 편집
- 맨 위 행: "new" 태그가 있는 입력 행 (새 접수)
- 기존 행: 클릭하면 해당 셀이 입력 필드로 변환 (구글시트처럼)
- 모달 없이 테이블에서 직접 입력/수정

#### 4.1.5 말풍선 아이콘 → 문자 패널
테이블 맨 왼쪽에 **말풍선 아이콘** 배치. 클릭하면 오른쪽에 **넉넉한 크기의 패널** (폭 420px 이상) 열림.

**패널 구성 (위→아래)**:
1. **고객 정보**: 이름, 연락처
2. **AS 정보 요약**: 입고일, 브랜드, 모델, 계산서, 증상, 입금상태
3. **진행 상황 바**: 접수→진단→수리중→출고→완료 (초록=완료, 노랑=현재)
4. **교체 부품 / 비용**: 부품 목록 + 최종 AS 비용
5. **문자 내역**: 카카오톡 채팅 형태 (파란말풍선=발신, 회색말풍선=수신)
6. **문자 입력창**: 하단에 input + 전송 버튼 → httpSMS로 실제 문자 발송

| AS연동 | 버튼/태그 | "연동"(AS건 연결) / "단독"(AS 없이) |

| 수령자명 | 텍스트 | |
| 수령자HP | 텍스트 | |
| 수령자 주소 | 텍스트 | |
| 우편번호 | 텍스트 | |
| 품목명 | 텍스트 | |
| 규격 | 텍스트 | |
| 수량 | 숫자 | |
| 품목비고 | 텍스트 | |
| 배송메시지 | 텍스트 | |
| 신용/착불 | 드롭다운 | 선불 / 착불 |

#### 4.2.3 필터
- 통합 검색 (수령자명, 연락처, 품목)
- 택배사 필터
- 송장 상태 (미입력 / 입력완료)
- 월별 선택

#### 4.2.4 송장 미입력 건
- 노란 배경으로 강조 표시
- 송장번호 셀에 초록 테두리 입력 필드

### 탭 3: 수리내역 조회

#### 4.3.1 통합 검색
- 화면 상단에 **검색창 1개** (16px 큰 폰트)
- 이름, 연락처, 거래처, 모델명, 증상, 처리결과, 운송장번호 — 어떤 키워드로든 검색

#### 4.3.2 고객 리스트
- 검색 결과에 해당하는 고객 목록 표시
- 각 항목: 아바타(성 이니셜) + 이름 + 연락처 + AS 건수 + 최근 날짜
- 클릭하면 아래에 상세 정보 펼침

#### 4.3.3 고객 정보 바
- 이름, 연락처, 거래처 여부
- 총 AS 건수, 총 비용, 보증중 건수

#### 4.3.4 수리 이력 테이블
| 컬럼 |
|------|
| 입고일, 브랜드, 모델, 증상, 처리결과, 처리자, 상태, AS비용, 출고일, 보증상태 |

**보증상태 자동 계산**:
- 새 제품 구매 후: **12개월** 이내 → "보증중" (초록)
- 동일 부속 AS 후: **6개월** 이내 → "보증중" (초록)
- 기간 초과 → "만료" (빨강)
- 보증 기간은 설정에서 변경 가능

#### 4.3.5 문자 내역 (수리 이력 아래에 배치)
- 해당 고객과 주고받은 문자를 **채팅 말풍선** 형태로 표시
- 파란 말풍선: 발신 (입고알림, 출고알림, 일반 답장)
- 회색 말풍선: 수신 (고객 답장)
- 날짜별 구분선

### 탭 4: 부속 가격

#### 4.4.1 검색 / 필터
- 통합 검색창 (부품코드, 품명, 스펙)
- 구분(모델) 드롭다운 필터
- 카테고리 칩: 모터/하우징, 메인보드, 가스켓/패킹, 액정/커버, 솔레노이드, 밸브/센서, 니쁠/피팅, 배기파이프, 기타

|------|------|

| 구분 (모델) | 뱃지 표시, "공용"은 파란 뱃지 |
| 공임비 | 오른쪽 정렬, 파란 볼드 |
| 관리 | 수정 버튼 |

- 하단에 합계 자동 계산

#### 4.5.2 시스템 설정 (자유 접근)

- 입고 알림 / 출고 알림 — 각각 편집 가능
- 변수 클릭하여 삽입: {입고날짜}, {출고날짜}, {모델명}, {택배사}, {운송장번호}, {고객명}, {거래처명}, {브랜드}, {AS비용}
- 입력창 높이: 내용 전체가 보이도록 auto-resize (스크롤 없이)

**SMS 연동 (httpSMS)**:
- API 키 입력 (password 타입)
- 발신 번호 (안드로이드 폰 번호)
- 연결 상태 표시 (연결됨/끊김)
- 테스트 문자 발송 버튼

**처리자 관리**:
- 처리자 이름 등록/수정/삭제
- 풀네임으로 등록 (예: 최영호, 김영수, 이준혁, 형진우)
- AS 일지에서 드롭다운으로 선택

**보증 기간 설정**:
- 새 제품 구매 시: 기본 12개월 (변경 가능)
- AS 후 동일 부속: 기본 6개월 (변경 가능)

**관리자 비밀번호 변경**:
- 현재 비밀번호, 새 비밀번호 입력

## 5. 문자(SMS) 기능

### 5.1 연동 방식
- **httpSMS** (httpsms.com) 사용
- 안드로이드 폰에 httpSMS 앱 설치 → 폰이 SMS 게이트웨이 역할
- 웹사이트에서 API 호출 → 안드로이드 폰이 실제 문자 발송
- 고객 답장 → httpSMS webhook → 웹사이트에 자동 수신/저장

### 5.2 자동 발송
- **입고 저장 시**: 입고 알림 문자 자동 발송 (토글 ON/OFF)
- **출고 저장 시**: 출고 알림 문자 자동 발송 (토글 ON/OFF)
- 템플릿은 설정에서 편집 가능

### 5.3 수동 발송
- AS 일지 말풍선 아이콘 → 문자 패널에서 직접 입력 후 전송
- 수리내역조회에서도 문자 내역 확인 가능

### 5.4 문자 기록
- 발신/수신 모든 문자는 Supabase에 자동 저장
- 고객 전화번호 기준으로 대화 내역 조회

## 10. 절대 하지 말 것

1. 기존 Supabase DB 데이터 삭제/초기화/변경
2. 요청하지 않은 파일/로직 수정
3. 디자인 시스템 외 임의 색상 사용
4. Pretendard 외 다른 폰트 사용
5. 섹션 헤더를 파란색으로 사용 (항상 #1A1D23 다크)
6. 금액을 K/M/B로 축약
7. 빌드 테스트 없이 push
8. API 키를 코드에 직접 입력 (환경변수 사용)
